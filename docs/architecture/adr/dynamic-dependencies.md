# ADR-002: Динамические зависимости

## Контекст
Разные модели и метрики требуют разных наборов Python-пакетов. Сборка отдельного Docker-образа под каждую комбинацию приводит к взрывному росту количества образов и затрат на хранение.

## Решение
Использование базовых образов с предустановленным `uv`. Специфичные зависимости устанавливаются "на лету" (Just-In-Time) при запуске контейнера с помощью команды `uv pip install` или `uv sync`.

## Последствия
*   **Плюсы**: Минимальное количество базовых образов, высокая гибкость, всегда актуальные версии пакетов из workspace.
*   **Минусы**: Требуется доступ в интернет при первом запуске (или локальный кэш/pypi mirror), небольшая задержка на старте контейнера.
