# ADR-006: Монорепозиторий и динамическая композиция

## Контекст
Проект состоит из множества независимых пакетов. Необходимо обеспечить их полную изоляцию при сохранении удобства разработки в едином воркспейсе.

## Решение
Отказ от `uv workspaces` в пользу **автономных пакетов** и **динамической композиции**. Каждый пакет в `packages/` имеет собственный `pyproject.toml` и `uv.lock`. Взаимодействие между ними в рантайме (внутри контейнеров) управляется через `DependencyInjector`.

## Последствия
*   **Плюсы**: Полная изоляция зависимостей, отсутствие конфликтов версий между модулями, возможность независимой публикации и переиспользования пакетов.
*   **Минусы**: Размер репозитория может расти, необходимость настройки CI для выборочного тестирования пакетов.
